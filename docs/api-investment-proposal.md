# 有料データAPI投資 提案書 — 担当者接続率・アポ数を定量で伸ばす

> 作成日: 2026-06-18 / 対象: 営業部門 上長・意思決定者
> 目的: テレアポの**担当者接続率**と**アポ数**を、外部データAPI投資でどれだけ伸ばせるかを**自社実測ファネルに基づく定量シミュレーション**で示し、投資判断の材料とする。
> 結論先出し: **月30万円（Plan B）の投資で、同じ架電工数のままアポ数が月21件→約57〜78件（2.7〜3.7倍）に伸びる見込み。増分アポ1件あたりのデータコストは約5千円で、受注価値（LTV約120万円）に対して桁違いに小さい。**

---

## 1. なぜ今、データに投資すべきか（自社実測が示す事実）

直近の架電 **4,886コール**の実測内訳：

| 結果区分 | 件数 | 比率 |
|---|---:|---:|
| 担当者不在 | 2,927 | **59.9%** |
| 担当者接触：お断り | 726 | 14.9% |
| 受付ブロック | 420 | **8.6%** |
| 鳴りっぱなし | 342 | 7.0% |
| コールのみ | 219 | 4.5% |
| 営業フォロー | 84 | 1.7% |
| 番号不備 | 52 | 1.1% |
| **担当者接触：アポ獲得** | **51** | **1.04%** |
| 現在使われていない | 30 | 0.6% |
| ヒアリング成功 | 26 | 0.5% |
| 問い合わせ・他 | 9 | 0.2% |
| **合計** | **4,886** | 100% |

ここから導かれる**現状KPI**：

- **担当者接続率 ＝ 17.6%**（アポ51＋お断り726＋営業フォロー84＝861 ÷ 4,886）
- **アポ率（対全コール）＝ 1.04%**
- **接続後アポ率 ＝ 5.9%**（51 ÷ 861）

### 診断：漏れの7割は「担当者に繋がっていない」ことが原因

アポに至らない最大要因は商談力ではなく、**そもそも担当者に接続できていない**こと。

- **担当者不在 59.9% ＋ 受付ブロック 8.6% ＝ 全体の68.5%** が「会社には届いたが本人に届いていない」状態。
- この層は **「採用担当者の個人名・直通／携帯番号」** があれば、名指し突破・直通接続・再コール精度で大きく回収できる。
- さらにアポ率1.04%という低さは、**「今まさに採用管理ツールを探している／求人出稿を増やしている」企業に絞れていない**ことが大きい。ここは**インテントデータ**が直撃する。

> **要点**：現状は「母集団は1,000社あるが、①本人に繋がらない ②今ホットな企業を狙えていない」という二重の取りこぼし。データ投資はこの2つの穴を**直接**塞ぐ。

## 2. 改善レバーは2つ。それぞれに効く外部データがある

| レバー | 何を改善するか | 効くデータ種別 |
|---|---|---|
| **レバー① 連絡先データ** | 受付ブロック突破・直通接続 → **担当者接続率↑** | 採用担当者個人名・直通／携帯・部署役職 |
| **レバー② インテントデータ** | 今ホットな企業に集中 → **接続後アポ率↑** ＋ リスト優先順位 | 求人出稿の増減・検索行動・採用ページ更新 |

**自社システム側は受け入れ準備が完了済み**（追加開発ほぼ不要）：
- 採点エンジン `quality.js` / `mochica-fit.js` は **`新卒出稿`・`出稿媒体数`・`予想出稿金額`・`採用担当者名` の列を最初から待っている**。CSVを社名／法人番号で結合するだけでスコアが「代理推定」→「実データ」へ自動昇格する。
- 全レコードに**法人番号**を保持済みのため、どのデータ提供元とも**名寄せがクリーン**。
- 効果測定の `source-kpi.js`（ソース別 接続→アポ→受注の利回り評価・2週間サイクル）も実装済みで、**投資効果を即座に数値で検証できる**。

→ つまり「データを買えば、その日から採点とKPI計測に乗る」状態。**投資の検証コストが極めて低い。**

## 3. 候補API 調査・比較

> 注：国内法人向けデータサービスの多くは料金が「要問い合わせ」。下表は**公開情報・比較サイト掲載のレンジ**であり、最終見積りは商談時に確定する。出典は文末。

| サービス | 主レバー | 公開料金レンジ | 採用領域への適合 | カバレッジ・留意点 |
|---|---|---|---|---|
| **HRogリスト（新着HOTプラン）** | ②インテント | **月3万円〜**（5媒体1エリア） | ◎ 採用特化。**求人出稿＝採用ニーズの実データ**。出稿金額予測あり | 100+求人サイトを毎週更新。マイナビ/リクナビ横断。**当社採点に設計済みで即接続** |
| **Sales Marker** | ②インテント＋①連絡先 | **月40万円前後〜** | ○ 「採用管理システム 比較」等の**検索行動**を直接捕捉＋キーマン・連絡先・CRM連携 | 汎用インテントだが顕在ニーズの精度は高い。初期導入支援込み |
| **Apollo.io** | ①連絡先（個人名・直通・携帯） | $49〜119/席・月（年額）。**実質$150〜400/席**（クレジット消費込み） | △ 個人名・直通の宝庫だが**日本のSME精度は60〜73%**（米国80〜88%より低い）。要トライアル検証 | 2.1億コンタクト。海外発のため日本中小カバレッジに注意 |
| **uSonar（ユーソナー）** | ①連絡先（名寄せ・電話付与） | **月10万円〜** | ○ 全国820万拠点LBCで**電話番号付与・名寄せ・属性補完** → 接続率改善 | データ統合（CDP）型。番号不備・名寄せ精度向上に効く |
| **Musubu** | 企業データ（安価な肉付け） | **月1.5万円〜**（12カ月20万円で24万クレジット） | ○ 140万社DB。電話・属性を**安価**に補完 | コスパ最良の土台。インテントは持たない |

**採用×中小×新卒という当社ターゲットでの優先度**：
1. **HRogリスト**（最優先）— 採用特化インテントで唯一「新卒含む出稿」を実データ化。月3万〜で最もコスパが高く、既存採点に即乗る。
2. **uSonar / Apollo**（接続率レバー）— 担当者名・直通の付与。Apolloは要トライアル（日本SME精度）。
3. **Sales Marker**（上振れ）— 検索インテントで「今検討中」を直撃。高額だが接続後アポ率の最大化に効く。
4. **Musubu**（土台）— 安価な企業データ補完。

## 4. 定量シミュレーション

### 4-1. 前提（透明化）

| 項目 | 値 | 根拠 |
|---|---|---|
| 月間架電数 | **2,000コール** | 現体制1〜2名（架電工数は固定＝追加人件費なしで比較） |
| ベース担当者接続率 | 17.6% | 自社実測 |
| ベース接続後アポ率 | 5.9% | 自社実測 |
| ベースアポ率 | 1.04% | 自社実測 |
| アポ→受注率 | **18%**（保守ケースは15%） | 採用管理SaaS×中小のインサイドセールス標準 |
| 受注1件の価値 | 年商48万円（月4万円×12）／LTV120万円（約2.5年） | ヒアリング（月3〜5万円×継続） |

> **アポ数 = 架電数 × 担当者接続率 × 接続後アポ率**。各APIがこの2つの率をどれだけ動かすかでアポ数が決まる。

### 4-2. 3プランと funnel 効果

| | **Plan A 点火**<br>月5万円 | **Plan B 本命 ★推奨**<br>月30万円 | **Plan C 最大化**<br>月70万円 |
|---|---|---|---|
| 構成 | HRog HOT（3万）＋Musubu（1.5万） | HRog（10万）＋Apollo 3席（〜12万）＋uSonar（10万） | Sales Marker（40万）＋HRog（10万）＋Apollo/uSonar（20万） |
| 主レバー | ②インテント中心 | ①連絡先＋②インテント両取り | ②検索インテント主導＋①連絡先 |
| 担当者接続率 | 17.6% → **19〜21%** | 17.6% → **25〜30%** | 17.6% → **28〜33%** |
| 接続後アポ率 | 5.9% → **8〜10%** | 5.9% → **10〜13%** | 5.9% → **14〜18%** |
| アポ率 | **1.5〜2.1%** | **2.5〜3.9%** | **3.9〜5.9%** |

### 4-3. アポ数・受注・ROI（月2,000コール固定）

| 指標 | ベース | Plan A（保守/期待） | Plan B（保守/期待） | Plan C（保守/期待） |
|---|---:|---:|---:|---:|
| **アポ数/月** | 21 | **30 / 42** | **50 / 78** | **78 / 119** |
| 増分アポ/月 | — | +9 / +21 | +29 / +57 | +57 / +98 |
| 増分受注/月（×18%） | — | 1.6 / 3.8 | 5.2 / 10.3 | 10.3 / 17.6 |
| 増分受注/年 | — | 19 / 45 | 63 / 123 | 123 / 212 |
| **増分年商（×48万）** | — | **約910万 / 約2,160万円** | **約3,000万 / 約5,900万円** | **約5,900万 / 約1.0億円** |
| 年間投資額 | — | 60万円 | 360万円 | 840万円 |
| **増分アポ単価**（データ費/増分アポ） | — | 約2,400円 | 約5,300円 | 約7,100円 |
| **初年度ROI**（増分年商÷投資, 期待） | — | 約36倍 | 約16倍 | 約12倍 |

> **保守ケース（アポ→受注15%）でも**：Plan B 増分受注 63件/年 × 48万 ＝ **約3,000万円/年** に対し投資360万円 → **ROI 約8倍**。下振れしても十分に投資回収可能。

### 4-4. 最も反論されにくい指標 ＝ 増分アポ単価

- **Plan B：増分アポ1件のデータコスト 約5,300円**。同じアポを今の架電で取るには、人件費換算で1件あたり数万円かかっている（2,000コール÷21アポ＝95コール/アポ）。
- **データは「アポ単価を下げる投資」**であり、人を増やすより安くアポを増やせる。
- 受注換算（保守15%）でも **1受注あたりデータコスト約3.5万円 vs LTV120万円 → 約34倍**。

## 5. 感度分析（前提が外れても成立するか）

Plan B（月30万）の**増分年商**を、2つの不確実性で振った表：

| アポ→受注率 ＼ 効果 | 保守（接続25%×アポ10%） | 期待（接続30%×アポ13%） |
|---|---:|---:|
| 12%（弱気） | 約2,000万円/年 | 約3,900万円/年 |
| 18%（標準） | 約3,000万円/年 | 約5,900万円/年 |
| 25%（強気） | 約4,200万円/年 | 約8,200万円/年 |

→ **どのセルでも投資360万円を大きく上回る**。投資が赤字になるのは「効果が現状から全く動かない（接続率17.6%のまま）」場合のみで、漏れ68.5%の構造を踏まえると非現実的。

## 6. 推奨：段階導入でリスクを最小化

**いきなり大型契約せず、効果実証→拡大の2段階を推奨。**

### フェーズ1（今月〜：Plan A 月5万円）
- **HRogリスト新着HOTプラン**を契約し、ホット企業（求人出稿中・出稿増）を当社採点に結合。
- 既存の `source-kpi.js`（2週間サイクル）で **HRog由来リードの接続率・アポ率を実測**。
- 投資5万円/月。**4-6週間で「データありリスト vs 現行リスト」のアポ率差を自社データで証明**する。

### フェーズ2（実証後：Plan B 月30万円）
- フェーズ1でリフトが確認できたら、**Apollo（連絡先トライアル）＋uSonar（電話名寄せ）**を追加し接続率レバーも稼働。
- 予算レンジ（月30〜100万）の下限で本命構成を回す。

### フェーズ3（上振れ判断：Plan C 月70万円）
- Plan Bの受注実績が出たら、**Sales Marker**で検索インテント（「採用管理システム 比較」検索企業）を起点化し最大化。

> **上長への提案骨子**：「まず月5万円でHRogを試し、2週間KPIで効果を数字で見せます。リフトが出たら月30万へ拡大。下振れしても感度分析の最悪セルで投資の5倍は回収します。」

## 7. リスクと留意点（先回り）

| リスク | 内容 | 対策 |
|---|---|---|
| 日本SMEの連絡先精度 | Apolloは海外発で中小の直通精度60〜73% | **トライアルで実精度を検証**してから席数拡大。国内はuSonar/HRogを主軸に |
| インテントのカバレッジ | HOTリストは母集団の一部。全2,000コールに高アポ率が乗るわけではない | シミュレーションは**ブレンド後の率**で算定済み（接続後アポ率を控えめに設定） |
| 効果の立ち上がり | データ投入初月から満額効果は出ない | フェーズ1で3ヶ月ランプを見込む。ROIは年換算で評価 |
| 高額プランの固定費 | Sales Markerは月40万の固定費 | Plan A/Bで実証してから。最初は契約しない |
| コンプライアンス | 個人情報・特定電子メール法・各媒体ToS | 公開API／許諾データに限定。電話は適正取得、メールはオプトイン運用を継続 |

## 8. まとめ（意思決定の論点）

1. **現状のボトルネックは商談力ではなく「担当者に繋がらない（68.5%）」「今ホットな企業を狙えていない（アポ率1.04%）」という構造**。自社実測が示している。
2. この2つは **連絡先データ（接続率）** と **インテントデータ（接続後アポ率）** で直接塞げる。当社システムは受け入れ準備完了済みで**追加開発ほぼ不要・検証コスト極小**。
3. **Plan B（月30万）でアポ数2.7〜3.7倍、増分アポ単価約5千円、初年度ROI約16倍（保守でも約8倍）**。
4. リスクは**月5万円のHRogから始め、2週間KPIで実証→拡大**する段階導入で最小化できる。

**意思決定のお願い**：まずフェーズ1（HRogリスト 月5万円）の承認をいただき、4〜6週間で効果を数値実証した上で、Plan Bへの拡大を再協議させてください。
